JavaScript秘密花园学习笔记

标签（空格分隔）： js

  • 除了null undefined外，所有变量都可以看成对象，调用对象的方法
  • hasOwnProperty()可以用于for in循环中作为if()判断条件 因为for in循环会枚举原型链上的所有属性 判断之后性能会好很多

  • arguments对象
    arguments不是数组，是一个object对象 为了方便使用一般转换为数组
    性能不是很好，比较慢的转换方法 Array.prototype.splice.call(arguments)
    推荐，将参数从一个函数传递到另一个的做法

  • 构造函数
    显示的return 表达式将会影响返回结果 但仅限于返回的是一个对象；
    return 2；不会影响 return new Number(2)就会返回数字对象
    如果new被遗漏了 则函数不会返回新创建的对象
    下面使用或者不使用new关键字都没有功能上的区别，这两种方式创建的对象都不能访问bar原型链上的属性

    • 作用域
      外部循环在第一次调用subloop之后就终止，因为sublop没有声明var i，i=10将外层循环中的i覆盖了

  • 变量声明
    js中变量只有两种申明方式：1是var声明 2是作为函数参数

  • 变量提升
    var声明和function都会变量提升 下面是变量提升的优秀案例

  • 数组的length变小会截短数组，但是变大不会改变数组

  • 类型typeof
    ==会强制转变变量类型 ===严格模式不会强制转变 这两个操作符都是等于操作符 但是当有一各操作数为对象时，行为就不同了。啫喱等于操作符比较的不是值是否相等，而是是否属于同一个身份，只有对象的同一个实例才被认为是相等的
    typeof唯一有用的地方，测试变量是否定义typeof foo !== "undefuned"
    typeof大部分返回object 为了检查对象的类型需要使用定义在object.prototype上的toString()方法

    instanceof 操作符应该仅仅用来比较来自同一个 JavaScript 上下文的自定义对象。 正如 typeof 操作符一样，任何其它的用法都应该是避免的.

  • 转换为布尔型

  • settimeout setinterval
    前者阻塞函数运行一会只执行一次，后者每xx秒执行一次
    setinterval：当函数被阻塞的时候，setInterval 仍然在组织将来对回调函数的调用。 因此，当第一次 函数调用结束时，已经有 10 次函数调用在等待执行。所以很容易造成阻塞，下面是最常用的方法处理阻塞。

  • 清除定时器

    var id = setTimeout(foo, 1000);
clearTimeout(id);

  clearTimeout(i);
